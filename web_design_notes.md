# Webデザイン原則の基礎 - 講座ノート

## 1. デザインの4原則（近接・整列・反復・対比）

「近接・整列・反復・対比」の4原則を意識するだけで、デザインのクオリティは劇的に向上する。

| 原則 | 内容 |
|------|------|
| **近接** | 関連する情報を近くにまとめて配置 → 情報のまとまりが直感的にわかりやすくなる |
| **整列** | 要素を見えない線で揃えて配置 → ページ全体に秩序と安定感を与える |
| **反復** | 色・フォント・レイアウトを繰り返し使用 → デザインに一貫性と安心感を与える |
| **対比** | サイズや色に強弱をつけて配置 → 重要な情報を際立たせ、視線を誘導する |

---

## 2. 余白の役割

- 余白は"単なる空きスペース"ではない
- 情報を整理し、デザインに**呼吸**を与える
- 字間・行間・要素周りの余白を適切にとると → 窮屈さがなくなり、**可読性が向上**

**NG例**：テキストがギュッと詰まっていて、文字間・行間が狭い  
**OK例**：適度に余白があるテキストブロック、行間が広めでスッキリ

---

## 3. 視線パターン

- ユーザーの視線は一定のパターンで動く
- Webでは「**F型**」「**Z型**」が代表的
- 視線が止まりやすい場所（左上・対角の終点など）に**重要情報を配置**すると効果的

---

## 4. 視覚階層（ビジュアルヒエラルキー）

ユーザーに「どこから見て、何が重要か」を瞬時に伝える技術。

```
重要度：大見出し > 小見出し > 本文 > 補足情報
```

### 視覚階層を作る方法
- フォントのサイズ → **大きいほど重要**
- フォントの太さ → **強調度合いを調整**
- 色 → **アクセントで重要要素を際立たせる**
- 配置 → **画面内の位置で注目度をコントロール**

### ファーストビューで伝えること
- ページを開いた瞬間に「何を伝えたいか」を明確に
- 大きな見出しで一番重要なメッセージを伝える
- サブ見出しや短い本文で補足
- 行動ボタン（CTA）を配置し、次のアクションを促す

---

## 5. 8の倍数ルール

サイズや余白を**8pxの倍数**（8, 16, 24, 32...）で設計するルール。

| 要素 | 推奨値 |
|------|--------|
| ボタン高さ | 32px |
| 見出しと本文の余白 | 16px |
| セクション間の余白 | 48px / 64px |

**なぜ8なのか？**
- 多くのデバイス解像度が8で割り切れる設計
- レイアウトが崩れにくく、どの画面でも見た目が整いやすい
- 余白のリズムがそろい、デザインに一貫性が生まれる

---

## 6. コンポーネント思考

- Webサイトを「**部品の集合体**」として設計する考え方
- ボタン・フォーム・カード型コンテンツなどを**再利用可能な部品**に分解
- レゴブロックのように組み合わせてページを構築

**メリット**
- 作った部品を使い回せる → **一貫性UP**
- 元の部品を直すだけ → **作業効率UP**

---

## 7. よくある失敗を防ぐチェックリスト

- [ ] 背景色と文字色のコントラストは十分か？
- [ ] 文字サイズは小さすぎないか？（本文は**16px以上**が目安）
- [ ] 不要な装飾で情報を邪魔していないか？
- [ ] 要素がバラバラに配置されていないか？
- [ ] 余白は均一に取れているか？

> **迷ったら「整列・コントラスト・余白」の3点を見直そう**
